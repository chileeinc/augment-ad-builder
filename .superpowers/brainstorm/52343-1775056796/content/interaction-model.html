What's the core interaction model?

This shapes everything else — pick the one that feels right for your team's workflow

A

Structured Form — Simplified

User fills in Headline, CTA, picks a platform and theme. The layout adapts automatically based on what's filled in (image present → product shot layout, stat present → stat layout, otherwise → big type). No template picker. No purpose picker. Just fill and preview.

Headline → "Augment makes devs 10% faster"
CTA → "Try free"
Theme → Dark
Platform → LinkedIn
▶ Preview updates live
Fast to use Predictable output Less creative variety
B

Prompt-Based — AI Generates Variations

User types a description like "Augment allows users to be 10% quicker at dev speed". AI parses this and auto-populates 3–4 design variations with different layouts, copy angles, and visual treatments. User picks the one they like and can tweak from there.

Prompt → "Augment makes devs 10% faster at code reviews"
▶ Generates 4 variations automatically
  [Big type: "10% Faster"] [Stat hero: 10%]
  [Quote: "dev speed..."] [Product shot]
Creative range Minimal input Needs AI API Less predictable
C

Hybrid — Prompt to Pre-fill Form

User can either fill the form directly OR paste a prompt. If they use a prompt, AI pre-fills the fields (headline, CTA, picks layout) but the user keeps full control to edit before exporting. Best of both — low friction start, high-control finish.

Prompt → "10% faster dev speed"
▶ Auto-fills: Headline, CTA, Layout
User edits fields if needed → Export
Fast start Full control Incremental build More to build
